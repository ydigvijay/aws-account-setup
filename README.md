# 📖 Project Overview

This project demonstrates how to create and secure an AWS account by following AWS best practices. It includes creating a new AWS account, configuring billing, verifying identity, enabling Free Tier alerts, and securing the root account with Multi-Factor Authentication (MFA).

This project is the first step toward learning AWS Cloud and building real-world cloud projects.

# ☁️ Step 1 – Creating My AWS Account

## What I Did

- Created a new AWS account.
- Verified my email address.
- Created a strong root user password.
- Successfully activated my AWS account.

### Steps Performed

1. Visited the AWS Sign-Up page.
2. Entered my email address and account name.
3. Verified my email using the verification code.
4. Created a secure root user password.
5. Completed the account creation process.

### Why This Step Is Important

An AWS account is required before using any AWS service. The Root User has complete administrative access, so it must always be protected with a strong password.

---

# 💳 Step 2 – Setting Up Billing & Payment

## What I Did

- Selected the AWS Free Plan.
- Added my personal contact information.
- Added my debit/credit card.
- Accepted the AWS Customer Agreement.

### Steps Performed

1. Selected the Free Plan.
2. Entered personal details.
3. Added payment information.
4. Accepted AWS Terms & Conditions.
5. Continued to account verification.

### Why AWS Requires a Payment Method

AWS requires a payment method to verify your identity and to charge only if you exceed the AWS Free Tier limits.

---

# ✅ Step 3 – Completing Account Activation

## What I Did

- Verified my mobile number using SMS.
- Completed identity verification.
- Selected the Basic Support Plan.
- Successfully activated the AWS account.

### Steps Performed

1. Entered my phone number.
2. Verified the SMS OTP.
3. Selected **Basic Support (Free)**.
4. Completed account activation.

### Why This Step Is Important

Identity verification prevents fraudulent account creation and helps secure AWS resources.

---

# 🔔 Step 4 – Enabling AWS Free Tier Alerts

## What I Did

- Logged into the AWS Management Console.
- Opened Billing & Cost Management.
- Navigated to Billing Preferences.
- Enabled AWS Free Tier Usage Alerts.
- Configured email notifications.

![Billing Alerts](http://nextwork.ai/confident_vermilion_quiet_allspice/uploads/aws-account-setup_j6k1l7m3)

### Steps Performed

1. Logged into AWS Console.
2. Opened Billing & Cost Management.
3. Clicked Billing Preferences.
4. Enabled AWS Free Tier Alerts.
5. Saved the changes.

### Why This Step Is Important

Free Tier alerts notify users when usage reaches approximately 85% of Free Tier limits, helping avoid unexpected charges.

---

# 🔐 Step 5 – Securing the AWS Account with MFA

## What I Did

- Opened the IAM Dashboard.
- Configured Multi-Factor Authentication (MFA).
- Connected an Authenticator App.
- Enabled MFA for the Root User.

![MFA](http://nextwork.ai/confident_vermilion_quiet_allspice/uploads/aws-account-setup_e3f7g2h8)

### Steps Performed

1. Opened IAM Dashboard.
2. Selected the Root User.
3. Chose Multi-Factor Authentication.
4. Connected the Authenticator App.
5. Verified the authentication code.
6. Successfully enabled MFA.

### Why This Step Is Important

MFA adds an additional security layer by requiring both a password and a verification code before allowing account access.
